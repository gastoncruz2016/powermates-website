Common questions

What source platforms do you support?
Azure Synapse Analytics (dedicated and serverless pools), Azure SQL Data Warehouse, and SQL Server. We can also assess other T-SQL-based platforms on a case-by-case basis.
Do I need Fabric capacity before we start?
Yes — you need an active Fabric capacity (F2+ SKU) with Warehouse workload enabled. We'll help verify during the discovery call.
What about stored procedures with unsupported T-SQL?
Our compatibility audit flags every unsupported feature with a workaround recommendation. During the working session, we rewrite the critical procedures together using Fabric-compatible patterns (MERGE, CTAS, incremental load).
Will my security policies carry over?
Yes. We systematically port Row-Level Security, Column-Level Security, and Dynamic Data Masking. The assessment identifies every security policy in your source so nothing gets missed.
What if I have more than 500 tables?
The engine handles any size. For very large schemas (500+ tables), we may recommend splitting into priority waves. The assessment scores each table so you know what to migrate first.
Can I run the assessment before committing?
We offer an assess-only mode. Contact us to discuss a lighter-weight assessment engagement before the full migration.
