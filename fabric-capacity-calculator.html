How we calculate this (and what we're not modeling)

This is a rule-of-thumb sizing tool, not a Microsoft-endorsed calculator. We built it because every prospect asks the same question and no one wants to run the Microsoft capacity calculator themselves.

The CU demand estimate adds up:

  • data volume CU — ranges from 2 (under 100 GB) to 60 (10 TB+)
  • user CU — ranges from 1 (< 25) to 32 (500+)
  • Multiplied by a refresh cadence factor (1.0 daily, 1.6 hourly, 2.4 streaming)
  • Plus Copilot CU (0 / 8 / 24) and mirroring CU (0 / 3 / 8)

We then apply a 20% safety buffer, match the result to the smallest Fabric SKU that covers it, and apply an F64 floor whenever Copilot is enabled (because Power BI Copilot requires F64+).

What we're not modeling: Direct Lake vs Import mix, semantic model size per workspace, query complexity, Spark pool cold-starts, Eventhouse ingestion throughput, and cross-region egress. All of these matter — which is why the tool gives you a range, not a guarantee, and why we built Tenant Scan to model them from your actual telemetry.

Prices are pay-as-you-go list prices in USD East US, April 2026. Reserved pricing assumes the standard 1-year commit (~29% off PAYG). Your actual negotiated rate may differ — Microsoft discounts are common above F128.
